Endpoints added:
  GET  /api/v1/handover/summary?shiftHours=8
    Response: HandoverSummary record with shift stats and AI advisory
    shiftHours parameter: 1-24, default 8
    Returns: totalAlerts, criticalAlerts, acknowledgedAlerts, openAlerts,
             topMachineIssues (up to 3), aiHandoverNote with confidence badge
    
  POST /api/v1/handover/note
    Body: { "operatorId": "OP-01", "note": "COMP-01 vibration elevated" }
    Response: { status, operatorId, message }
    Persists to S3: handover/{date}/note_{epoch}.json
    
  GET  /api/v1/handover/notes?date=2026-03-22
    Response: List<String> of JSON note objects for the given date
    Date format: YYYY-MM-DD (defaults to today)
    Loads from S3: handover/{date}/ prefix

Storage:
  Alert queries: DynamoDB scan with timestamp filter (GSI recommended for production)
  Operator notes: S3 with structure: handover/{YYYY-MM-DD}/note_{epoch}.json
  ADR-004 compliant: DynamoDB for alerts (operational), S3 for documents

AI output:
  AiAdvisoryWrapper pattern: Map<String, Object> with advisory metadata
  Confidence: HIGH if totalAlerts > 5, MEDIUM otherwise
  Risk score: ELEVATED if critical alerts present, NORMAL otherwise
  All advisory text labelled: "[AI ADVISORY | Not a control action | ...]"
  Valid for: 30 minutes
  ADR-003 compliant: all LLM/generated text carries advisory wrapper
